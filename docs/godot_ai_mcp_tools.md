# Recommended Tools for Godot Development & AI MCP Integration

This document serves as a reference for tools, plugins, extensions, and workflows that enhance Godot 4 development—especially when combining terminal environments (e.g., Godot XTerm) with AI Assistant MCP (Model Context Protocol) capabilities.

---

## 1. Visual Debugging in Terminal Environments

Even when working inside terminal-focused setups or `godot-xterm`, visual debugging can be seamlessly integrated:

* **Image & Render Captures (`generate_image` & Media Inspection):** AI tools can directly view and analyze PNG, JPG, SVG, and render output files generated by Godot viewport captures.
* **In-Engine Remote Viewport Snapshots:** Custom debug helper scripts or autoloads can dump framebuffers, camera outputs, collision shape debug drawings, or ASCII shader passes directly to disk.
* **Command-Line Screenshot Generation:**
  ```bash
  godot --headless --path . --script res://addons/debug_screenshot.gd
  ```
  This enables automated visual snapshot captures during headless test execution.

## 2. Godot MCP Servers & Plugin Ecosystem

Because this project utilizes a custom TCP server ([mcp_interaction_server.gd](file:///home/jammy/ascii-midnight-citadel-engine/mcp_interaction_server.gd)), integrating standard Godot MCP servers further unlocks AI inspection:

* **Godot MCP Server (`godot-mcp` / `mcp-godot`):**
  * Standardized MCP server allowing AI assistants to query the Godot Editor via local WebSockets or JSON-RPC.
  * **Key Capabilities:** Read scene node trees (`get_nodes`), inspect resources (`.tres`), trigger build/run commands, check active scenes, and stream debug logs.
* **Godot Editor AI Plugins (AssetLib):**
  * `Godot-AI-Assistant` or `Godot-Copilot` plugins for inline LLM auto-completion inside Godot's built-in script editor.

## 3. Visual & Rendering Inspection Tools

* **RenderDoc / RenderDoc CLI:**
  * Industry-standard graphics frame debugger.
  * Useful for inspecting custom post-processing shaders, ASCII render pipelines, uniform buffer states, and draw calls from the command line.

## 4. Terminal Linters & Parser Utilities

* **`gdtoolkit` (`gdlint` & `gdformat`):**
  * Python-based GDScript linter and formatter targeting Godot 4.
  * Installed in project virtual environment (`./.venv/bin/`).
  * Command-line usage:
    ```bash
    # Format files
    ./.venv/bin/gdformat path/to/file.gd
    ./.venv/bin/gdformat --check path/to/file.gd

    # Lint files
    ./.venv/bin/gdlint path/to/file.gd
    ```
* **GUT (Godot Unit Testing - v9.7.1 for Godot 4.7):**
  * Installed in `addons/gut/`.
  * Headless test execution command:
    ```bash
    godot --headless -s addons/gut/gut_cmdln.gd -gdir=res://test/ -gexit
    ```
* **`godot-cli` / `godot-parser`:**
  * Utility scripts for syntax-checking `.tscn`, `.gd`, and `.tres` files off-thread before launching full engine instances.
